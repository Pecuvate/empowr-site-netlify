# Agents

This file defines the agents involved in this project. In context engineering, an agent is any autonomous or semi-autonomous process that takes instructions, uses tools, and produces outputs — including Claude itself when operating in a scoped role.

## What belongs here

- **Agent roles** — what each agent is responsible for and what it is NOT responsible for
- **Capabilities** — what tools, APIs, or integrations each agent can use
- **Boundaries** — decisions the agent must not make autonomously; things it must always confirm with a human
- **Inputs and outputs** — what each agent receives and what it returns
- **Inter-agent interactions** — if multiple agents exist, how they hand off work to each other

## Why this matters

Ambiguous agent scope leads to agents overreaching, duplicating work, or leaving gaps. Defining roles here means every session starts with a clear contract for what Claude is and isn't doing in this project.
