# Agent Roles

This file defines the agents involved in this project. In context engineering, an agent is any autonomous or semi-autonomous process that takes instructions, uses tools, and produces outputs, including Claude itself when operating in a scoped role.

## What Belongs Here

- Agent roles: what each agent is responsible for and what it is not responsible for
- Capabilities: what tools, APIs, or integrations each agent can use
- Boundaries: decisions the agent must not make autonomously
- Inputs and outputs: what each agent receives and returns
- Inter-agent interactions: how multiple agents hand off work

## Why This Matters

Ambiguous agent scope leads to agents overreaching, duplicating work, or leaving gaps. Defining roles here gives each session a clear contract for what Claude is and is not doing in this project.
